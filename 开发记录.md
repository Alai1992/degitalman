# 抗美援朝老兵数字人项目 - 完整开发记录

## 项目概述

本项目是一个抗美援朝老兵数字人展示网站"云寻老兵烽火迹"，旨在纪念抗美援朝战争74周年，通过数字人技术重现老兵风采，传承红色精神。

**项目名称**：抗美援朝老兵数字人
**技术栈**：Vite + TypeScript + Express + Tailwind CSS
**访问域名**：${COZE_PROJECT_DOMAIN_DEFAULT}
**项目目录**：/workspace/projects

---

### 2.1 主页模块按钮图标调整

**用户原始需求**：主页按钮上的小图标缩小，放在左上角

**问题分析**：默认状态下，模块按钮上的图标是100%宽度，且居中显示在按钮内部。用户希望图标缩小并移动到左上角位置。

**修改文件**：`/workspace/projects/src/index.css`

**修改前的代码**：
```css
.module-icon {
  width: 100%;
  position: absolute;
  top: auto;
  right: auto;
  /* ... 其他样式 */
}
```

**修改后的代码**：
```css
.module-icon {
  width: 50px;              /* 从100%改为固定50px */
  position: absolute;       /* 保持绝对定位 */
  top: 8px;                 /* 从auto改为8px */
  left: 8px;                /* 从auto改为8px，右上角改为左上角 */
  /* ... 保持其他样式 */
}
```

**修改效果**：
- 图标尺寸：从占据整个按钮宽度缩小为固定50px
- 图标位置：从居中显示改为左上角（top:8px, left:8px）
- 保持了金色阴影和发光效果

#### 第四阶段：功能完善（音频互斥播放）

**用户反馈**：播放《志愿军战歌》时，无法停止《英雄赞歌》的播放

**问题分析**：两个音频播放器相互独立，没有统一管理

**解决方案**：
```javascript
// 统一管理音频状态
let isPlaying = {
  song: false,
  hero: false
};

// 当前正在播放的音频
let currentAudio = null;

// 切换音频播放
function toggleAudio(type) {
  if (type === 'song') {
    // 如果英雄赞歌在播放，先停止
    if (isPlaying.hero && currentAudio === 'hero') {
      stopAudio('hero');
    }
    // 播放志愿军战歌
    const audio = document.getElementById('audioPlayer');
    audio.play();
    isPlaying.song = true;
    currentAudio = 'song';
    updateButton('song', '■ 停止');
  } else {
    // 如果战歌在播放，先停止
    if (isPlaying.song && currentAudio === 'song') {
      stopAudio('song');
    }
    // 播放英雄赞歌
    const audio = document.getElementById('heroAudioPlayer');
    audio.play();
    isPlaying.hero = true;
    currentAudio = 'hero';
    updateButton('hero', '■ 停止');
  }
}

// 停止指定音频
function stopAudio(type) {
  const audio = type === 'song' 
    ? document.getElementById('audioPlayer')
    : document.getElementById('heroAudioPlayer');
  audio.pause();
  audio.currentTime = 0;
  isPlaying[type] = false;
  if (currentAudio === type) currentAudio = null;
  updateButton(type, '▶ 播放');
}
```

### 2.5 问题修复

#### 问题1：子页面返回时tab状态丢失

**用户原始需求**：返回知识宝库页面时，展示的栏位为之前选择的栏位

**问题分析**：
- 用户点击"历史地图" → 进入子页面 → 点击返回 → 应该显示"历史地图"tab
- 但之前没有传递参数，返回后默认显示第一个tab

**解决方案**：使用URL参数传递来源

**修改所有子页面**：
```javascript
// map-korea-terrain.html 中
function goBack() {
  const params = new URLSearchParams(window.location.search);
  const from = params.get('from') || 'audio';
  location.href = `learning-support.html?tab=${from}`;
}
```

**修改知识宝库页面**：
```javascript
// 页面加载时检查URL参数
window.onload = function() {
  const params = new URLSearchParams(window.location.search);
  const tab = params.get('tab');
  if (tab) {
    switchTab(tab);
  }
};
```

**修改文件**：
- `/workspace/projects/public/learning-support.html`
- `/workspace/projects/public/map-korea-terrain.html`
- `/workspace/projects/public/map-five-campaigns.html`
- `/workspace/projects/public/war-diary.html`

## 四、开发方法总结

### 4.1 前端开发流程
1. 使用 `read_file` 读取现有代码，理解结构
2. 使用 `grep_file` 快速定位需要修改的代码位置
3. 使用 `edit_file` 修改关键部分
4. 使用 `test_run` 中的 `curl` 验证页面响应

### 4.2 资源下载流程
1. 使用 `generate_image` 生成AI图片
2. 使用 `curl -L -o` 下载用户提供的文件
3. 保存到 `/workspace/projects/public/` 目录
4. 验证文件：`ls -la` 和 `curl -I`

### 4.3 调试技巧
- 修改后立即用 `curl -I` 验证页面返回200
- 使用 `ls -la` 检查文件是否正确保存
- 使用 `grep_file` 的 `-A` 和 `-B` 参数查看上下文

### 4.4 常见问题处理
- **音频加载慢**：使用本地文件而非在线链接
- **图片标注重复**：精简AI生成提示词
- **tab状态丢失**：使用URL参数传递
- **音频播放冲突**：统一管理播放状态
